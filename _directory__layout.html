Directory Layout

Directory layout

/                       // Root dir which is the working directory for the executables so log files etc will be placed here.
                        // Also contains the Visual Studio DC.sln file which is used to build DC.
/cleanup.bat            // A Windows batch file used to delete any temporary build files. Usefull if we're backing this entire project to reduce size.
/DC.sln                 // A Microsoft Visual studio solution file which we can load into Visual studio to build everything.
                        // After a batch build which includes release builds, the compiled executables are placed in the root directory via Post build events.
/Data/                  // Files used by the compiled executables such as textures, models etc.
/Data/Shared/           // Files which are used by multiple projects.
/Doxygen/               // Files used and created by Doxygen, including the Doxygen configuration and executable files.
/Doxygen/Images/        // Various image files used when generating the Doxygen documentation.
/Doxygen/Output/        // Where the Doxygen generated html output files are located.
/Doxygen/Pages/         // Various .md files used to flesh out the Doxygen generated documentation.
/SOMELIB/               // A directory containing a Visual studio project file along with the source code used to build a static library.
/SOMELIBExample/        // A directory containing a Visual studio project file along with the source code used to build an executable which is used to demonstrate/test a static library.
/ThirdParty/            // Projects by other people which the DC project needs, each placed inside their own sub-directory.
